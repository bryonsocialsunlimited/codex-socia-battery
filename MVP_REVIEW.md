# Social Battery MVP Shell Review

## Missing from the first-pass shell

- Rich seed data for users, bookings, memberships, ledgers, conversations, and admin flags
- Clear group risk states for below-3 merge decisions
- Feedback and become-friends review signals
- Membership and pricing plan examples beyond individual outing prices
- Reusable specialized cards for groups, rewards, pricing, memberships, and admin alerts
- Stronger mobile-app motion and a more cohesive premium visual system
- Backend-ready route structure for future Supabase, Stripe, realtime messaging, grouping, and admin metrics
- More complete admin review queues

## Still intentionally not built

- Real Supabase auth, database writes, storage, or row-level security
- Real Stripe checkout or billing
- Real group chat or direct messaging
- Automated ticket purchasing
- Automated identity verification
- Full grouping engine, feedback submission, or friendship persistence

## Current prototype improvement

The app now has richer mock data, reusable card components, subtle motion, clearer admin queues, reward/credit/membership UI, mock conversations, below-3 group states, and mock API route placeholders. It remains a polished clickable prototype rather than a production backend.
